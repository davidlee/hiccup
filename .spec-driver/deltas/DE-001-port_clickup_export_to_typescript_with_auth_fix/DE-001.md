# DE-001 – Port clickup-export to TypeScript with auth fix

## 1. Summary & Context

- **Implementation Plan**: [IP-001](./IP-001.md) – draft
- **Change Drivers**: Org language alignment (TypeScript); broken ClickUp API auth causing abort on permission-restricted endpoints

## 2. Motivation

- The current Python implementation aborts on ClickUp API permission errors (HTTP 403/401 on certain endpoints). When the API token lacks access to a subtask or resource, the tool crashes instead of gracefully degrading.
- TypeScript is the org-standard language; a Python dependency is friction for the team.
- The subtask-fetching approach makes redundant API calls (re-fetches data already returned by the list endpoint) and lacks pagination.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - TypeScript CLI that exports ClickUp task hierarchies to JSON
  - Graceful handling of permission-denied responses (skip inaccessible resources, warn, continue)
  - Correct tree assembly from flat task list (no redundant API calls)
  - Pagination support for lists >100 tasks
- **Operational Constraints**: No existing users; free to change CLI interface, output format, or API version as needed

## 4. Out of Scope

- UI or interactive features
- Polishing beyond a useful, correct v1

## 5. Approach Overview

- **System Touchpoints**: New TypeScript package (Node.js), replaces `clickup_export.py`
- **Key Changes**:
  - Rewrite in TypeScript with proper error boundaries around API calls
  - Build tree from flat list response using `parent` field (no N+1 fetches)
  - Add pagination loop for list endpoint
  - Handle 401/403 per-request: log warning, skip resource, continue export
- **Migration / Rollout Notes**: None — no users, clean break from Python version. Use v3 API if it's better suited.

## 6. Verification Strategy

- **Acceptance Criteria**:
  - CLI runs against a real ClickUp list and produces valid tree.json + task files
  - Permission-denied on a subtask logs a warning and does not abort
  - Lists with >100 tasks export completely (pagination)
  - Output is a sane, usable JSON hierarchy
- **Planned Artefacts**: VT-001 (unit tests), VH-001 (manual run against live ClickUp)

## 7. Risks & Mitigations

- **Risk**: ClickUp API response shape differs from assumptions – _Likelihood_: medium – _Impact_: medium – _Mitigation_: Reference API docs + existing Python code; test against live API early
- **Risk**: Permission errors manifest as shapes other than 401/403 – _Likelihood_: low – _Impact_: low – _Mitigation_: Catch all non-2xx per-request, log and continue

## 8. Follow-ups & Tracking

- **Future Deltas**: Polish pass after v1 is validated against real data
- **Open Decisions**: Package manager choice (npm vs pnpm) — defer to org convention

## 9. Implementation Notes

- ClickUp API v2 docs: reference `docs/v3.yaml` in repo (despite filename, covers v2 endpoints used)
- Existing `.envrc` exposes `CLICKUP_API_TOKEN`
